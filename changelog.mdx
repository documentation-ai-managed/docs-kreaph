## Recent Releases

Kreaph releases follow semantic versioning. You receive automatic notifications for major updates through the dashboard. Review changes below to understand new capabilities and migration steps.

<Callout kind="info">
  Follow [@kreaph](https://github.com/kreaph/kreaph) on GitHub for instant release alerts and participate in discussions.

<Update label="2024-11-15" description="v1.3.0" tags={["feature", "improvement"]}>

## New Features

- Added real-time collaboration editing for documentation pages. Multiple users now edit simultaneously with live cursors and conflict resolution.
- Introduced version history with diff viewer. Restore previous versions or compare changes visually.

## Improvements

- Enhanced search with fuzzy matching and full-text indexing, reducing query time by `>50%`.
- Optimized image uploads supporting WebP and AVIF formats automatically.

## Bug Fixes

- Fixed markdown rendering issues with nested lists and tables.
- Resolved authentication token refresh loops in long sessions.

To upgrade:

````bash
npm install kreaph@latest
````

</Update>

## How to Stay Updated

<Steps>
  <Step title="Subscribe to Releases" icon="bell">
    Visit the [GitHub repository](https://github.com/kreaph/kreaph/releases) and click "Watch" > "Releases only".
  </Step>
  <Step title="Enable Notifications" icon="settings">
    In your Kreaph dashboard, toggle email alerts under Account > Notifications.
  </Step>
  <Step title="Check API Version" icon="code">
    Query `https://api.example.com/v1/version` to verify your instance.
  </Step>
</Steps>

<Expandable title="Migration Between Major Versions" default-open="false">

When upgrading from v1.x to v2.0, update your `{API_KEY}` and reconfigure webhooks.

```javascript
// Old v1 auth
fetch('https://api.example.com/docs', {
  headers: { Authorization: `Bearer ${oldToken}` }
});

// New v2 auth
fetch('https://api.example.com/v2/docs', {
  headers: { 'X-API-Key': `your-api-key-here` }
});
```

</Expandable>
